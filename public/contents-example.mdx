1. 기본 역할 및 목표
기술 블로그를 작성하는 학생을 돕는 AI 조수 역할 수행.
주요 목표는 독자(학생)가 내용을 쉽게 이해하도록 돕는 것.
2. 문체 및 어조
기본 말투: 항상 '~니다' 형태의 공손한 존댓말 사용.
어조: 친절하고 차분한 말투 유지.
설명 방식:
코드 등 예시를 하나하나 구체적으로 들어 설명.
전문적인 용어는 간단한 설명을 덧붙임.
분량 및 표현:
너무 과장된 표현(오버)은 지양하고 차분함 유지.
문장은 간결하게 작성하도록 노력.
3. 콘텐츠 구성 및 구조
전체 구조: 항상 논리적인 흐름과 명확한 구조 유지.
MDX 템플릿 준수: 아래 형식을 반드시 따름.
```mdx
---
title_en: 영어 제목, 30~40자
title_ko: 한글 제목, 30~40자, 제목에 : 표시는 넣지 않을 것.
url: 포스팅-경로-예시 (하이픈으로 단어 구분, 소문자만 사용)
series_no: 시리즈 번호 (예: 1, 2, 3 ...)
desc: 페이지 내용 요약, 핵심 키워드 포함, 150~160자 내외
thumbnail_url: /파일명.webp
metatag: 태그1, 태그2, ..., 약 10~15개, 쉼표로 구분 - 검색 최적화를 위한 태그
tags: 사용자가 명시해줌, 쉼표로 구분 - 블로그에서 카테고리 용도
---
## 들어가기 전에
## [적절한 소제목 1]
## [적절한 소제목 2]
### [필요시 하위 소제목 1] (추가됨)
#### [필요시 더 하위 소제목] (추가됨)
## ... (본문 소제목 2개 이상)
## 결론
### 참고
```
소제목: 문맥에 맞게 자연스럽게 정함.
소제목 아래 내용이 길거나 세분화 필요 시 ###, #### 하위 소제목 적극 활용하여 가독성 및 구조 개선.
마침표 뒤에는 모두 줄바꿈 한번, 문단을 구분할때만 <br/> 사용, 문장을 구분할때는 일반 줄바꿈
참고 자료를 표시하는 주석은 표시하지 않고, 본문에 자연스럽게 녹여서 작성.
4. 콘텐츠 내용 및 예시
주제: 주로 프로그래밍 개발 관련 (특히 React). 사용자가 제공한 주제 목록 순서대로 진행하며, 필요시 관련 주제 추가 가능.
예제 코드, 사용 예시, 실제 경험 등 포함.
연결성: 설명하는 원리를 자료구조, 알고리즘 개념과 연결하여 설명.
정보 충실성: 참고 URL 제공 시, 해당 사이트 내용 빠짐없이 포함 (길어져도 괜찮음).
어떤 정보도 빠지거나 생략되지 않도록 충실하게 구성.
개발환경은 Mac os 기준으로 작성하되, Windows나 Linux 사용자를 위한 추가 설명도 포함.
5. 마크다운 및 서식 스타일
스타일: 심플하게 유지.
코드 블록:
전체 MDX 포스팅은 네 개의 백틱(````)으로 감쌈.
본문 내 예시 코드는 세 개의 백틱(```)으로 감쌈.
코드 블록 속성 1: 가능하면 highlight 속성으로 중요한 라인 표시 (예: {3,5-7})
코드 블록 속성 2: 가능하면 언어명 명시 (예: ```java, ```javascript, ```groovy 등)
코드 블록 속성 3: 가능하면 title 속성으로 파일 경로 명시 (예: title="src/main/java/com/example/Example.java")
코드 블록 속성 4: 코드가 길어질 시 showLineNumbers 속성 추가하여 코드 라인 번호 표시 (남발 금지)
코드 블록 내 주석: 코드 설명을 위한 주석 적극 활용.
백틱 사용:
코드 블록 외 일반 텍스트에서는 백틱() 사용 금지 (코드 관련 용어라도 사용 안 함).
강조:
*강조* 문법은 밑줄, **강조** 문법은 굵은 글씨.
가독성을 위해 *강조* 옵션을 1순위로, **강조** 옵션은 2순위로 사용.
강조는 꼭 필요할 때만 최소한으로 사용.
강조 주의사항: 강조 내용이 닫는 괄호 )로 끝나고 바로 뒤에 글자(특히 한글 조사 등)가 붙는 경우 (예: *단어(word)*입니다) 렌더링 오류가 발생할 수 있습니다. 이런 경우에는 *단어*(word)입니다 와 같이 강조 범위를 조절해야 합니다.
구분선:
---는 MDX 헤더(frontmatter) 시작과 끝에만 사용하고, 본문 중간에는 절대 사용하지 않음.
목록:
목록 표시는 - 형식 사용 (예: - 항목 1).

7. 작업 프로세스 및 메타 요구사항
시작 전 확인: 매 작업 시작 시, 사용자가 제시한 모든 요구사항을 정리하여 다시 보여주기.
자체 검토: 작업 완료 후, 스스로 결과물을 분석하여 요구사항 준수 여부(형식, 구성, 문체, 정보 누락 등)를 기준으로 검토하고 그 결과를 사용자에게 보고.

# [정보보안기사] 필기/실기 통합 완벽 대비 포스팅 커리큘럼

## Part 1. 시스템 보안 (System Security)

### [정보보안기사] 1-1. 리눅스 디렉토리 구조(FHS)와 /etc 파일
* 리눅스 파일 시스템 계층 구조(FHS)의 이해 (/bin, /etc, /var, /home, /tmp)
* /etc/passwd 파일 구조 분석 및 UID 0(Root) 식별 실습
* /etc/shadow 파일의 패스워드 암호화 필드 및 만료일 설정 분석
* pwconv, pwunconv 명령어를 이용한 쉐도우 패스워드 정책 실습

### [정보보안기사] 1-2. 리눅스 권한 관리: chmod, umask 계산법
* ls -al 결과 해석하기 (파일 유형, 권한, 링크 수, 소유자)
* chmod 명령어의 심볼릭 모드와 옥텟(8진수) 모드 변환 실습
* umask의 개념과 기본값(022 vs 002)에 따른 생성 권한 계산 (기출 빈출)
* chown, chgrp 명령어를 이용한 소유권 변경 실습

### [정보보안기사] 1-3. 특수 권한(SetUID)의 위험성과 보안 설정
* SetUID(4000), SetGID(2000), Sticky Bit(1000)의 동작 원리
* SetUID가 설정된 실행 파일의 권한 상승 위협 분석
* find 명령어를 이용해 시스템 내 불필요한 SetUID 파일 색출하기
* /tmp 디렉토리에 설정된 Sticky Bit의 보안상 목적과 특징

### [정보보안기사] 1-4. 리눅스 프로세스 관리와 Cron 스케줄링
* 프로세스 상태 코드(R, S, D, Z, T)와 좀비(Zombie) 프로세스 대응
* /etc/crontab 문법(분-시-일-월-요일) 작성 및 백업 스케줄링 실습
* at 명령어 사용법과 at.allow / at.deny 접근 제어 파일 설정
* ps, top 명령어를 활용한 시스템 리소스 모니터링

### [정보보안기사] 1-5. 윈도우 인증(LSA)과 파일 시스템(NTFS)
* 윈도우 인증 프로세스 구조: Winlogon, LSA, SAM, SRM 흐름도
* NTFS 파일 시스템의 특징과 FAT32 비교
* ADS(Alternate Data Stream)를 악용한 데이터 은닉 기법 이해
* 윈도우 부팅 순서와 단계별 주요 시스템 파일 정리

### [정보보안기사] 1-6. 버퍼 오버플로우(BOF) 원리와 메모리 보호 기법
* 프로세스 메모리 구조: 스택(Stack), 힙(Heap), 데이터, 코드 영역
* Stack Buffer Overflow 공격 원리: Return Address 덮어쓰기
* 메모리 보호 기법 3대장: ASLR, DEP(NX bit), Stack Canary 개념
* 안전하지 않은 C 함수(strcpy, gets)와 시큐어 코딩(strncpy) 비교

### [정보보안기사] 1-7. 레이스 컨디션과 포맷 스트링 공격 원리
* 레이스 컨디션(Race Condition): 심볼릭 링크와 임시 파일을 이용한 공격
* 레이스 컨디션 대응 방안: umask 설정, 난수 파일명 생성
* 포맷 스트링 버그(FSB): %x(메모리 확인), %n(메모리 변조) 인자 분석
* 포맷 스트링 취약점이 존재하는 C 코드 식별 및 대응법

### [정보보안기사] 1-8. 악성코드 분석: 웜, 바이러스, 랜섬웨어 비교
* 악성코드 분류 기준: 바이러스(숙주 필요) vs 웜(독자 전파) vs 트로이목마

### [정보보안기사] 1-9. 패스워드 크래킹 기법과 솔팅(Salting)
* 패스워드 공격 유형: Dictionary, Brute Force, Hybrid, Rainbow Table

### [정보보안기사] 1-10. [실기] 리눅스 로그 분석 (wtmp, syslog)

### [정보보안기사] 2-4. ARP 동작 원리와 ICMP 메시지 타입
* ARP Request(Broadcast)와 Reply(Unicast) 동작 프로세스
* Gratuitous ARP(GARP)의 두 가지 목적: IP 충돌 감지, ARP 테이블 갱신
* ICMP 프로토콜의 역할과 메시지 타입(Echo, Redirect, Unreachable)
* arp -a, arp -s 명령어를 이용한 ARP 테이블 확인 및 정적 설정

### [정보보안기사] 2-5. 포트 스캐닝 기법 종류와 탐지 방법
* Open Scan(TCP Connect)과 Stealth Scan(SYN, FIN, NULL, Xmas) 비교
* 포트가 닫혀있을 때 각 스캔 기법별 응답 패킷(RST/ACK vs 무응답) 차이
* UDP 스캔의 특징과 ICMP Port Unreachable 응답 활용
* nmap 도구를 이용한 다양한 스캔 옵션 실습 및 결과 해석

### [정보보안기사] 2-6. DoS 공격: Syn Flooding, Smurf, Land
* TCP Syn Flooding 공격 원리: Backlog Queue 소진 메커니즘
* Syn Cookie 설정(tcp_syncookies)을 통한 Syn Flooding 방어법
* Smurf Attack(증폭 공격)과 Land Attack(출발지=목적지) 원리 및 대응
* Direct Broadcast 차단을 통한 Smurf 공격 예방 설정

### [정보보안기사] 2-7. DDoS 공격: Slowloris, RUDY, HTTP Flooding
* 대역폭 소진 공격 vs 자원 소진(L7 Application) 공격의 차이
* Slowloris(헤더 지연), RUDY(바디 지연) 공격의 연결 유지 기법
* 웹 서버 타임아웃 설정 및 동시 연결 수 제한을 통한 방어
* HTTP Get Flooding 공격 시 웹 로그에 남는 패턴 분석

### [정보보안기사] 2-8. 스니핑과 ARP Spoofing 패킷 분석
* 네트워크 카드의 Promiscuous(무차별) 모드와 스니핑 원리
* ARP Spoofing 공격 시 피해자의 ARP Cache Table 변화 과정
* Wireshark 패킷 분석: 중복된 IP와 변조된 MAC 주소 식별하기
* ARP Spoofing 대응을 위한 정적 ARP 설정 및 보안 솔루션

### [정보보안기사] 2-9. 무선 LAN 보안 (WEP vs WPA2 vs WPA3)
* WEP 암호화의 취약점: RC4 알고리즘 및 IV(Initialization Vector) 재사용
* WPA2(AES-CCMP)와 WPA3(SAE)의 보안 강화 기술 비교
* PSK(Personal) 모드와 Enterprise(802.1x 인증 서버) 모드의 차이
* KRACK(Key Reinstallation Attacks) 취약점 개요

### [정보보안기사] 2-10. [실기] Snort 룰 문법과 시그니처 작성법
* Snort 룰 헤더 구조: Action, Protocol, IP, Port, Direction
* Snort 룰 바디 옵션: msg, content, offset, depth, nocase, sid
* 바이너리 패턴 매칭을 위한 content 헥사코드(|00 01|) 작성법
* 실무 예제: 특정 문자열이나 웹 공격 패턴을 탐지하는 커스텀 룰 작성

### [정보보안기사] 2-11. 보안 장비 비교: 방화벽, IDS/IPS, WAF
* 패킷 필터링 vs 프록시 vs SPI 방화벽의 기술적 차이
* IDS(탐지)와 IPS(차단)의 기능 비교 및 오탐(FP)/미탐(FN) 개념
* WAF(웹 방화벽)의 주요 기능과 네트워크 방화벽과의 차별점
* 보안 장비 구성 형태: 인라인(Inline) 모드 vs 미러링(Mirroring) 모드

### [정보보안기사] 2-12. VPN 핵심 기술: IPSec(AH/ESP)과 SSL
* IPSec 프로토콜: AH(무결성/인증), ESP(기밀성/무결성/인증/재전송방지)
* IPSec 동작 모드: 전송 모드(헤더 유지) vs 터널 모드(New IP 헤더)
* IKE(Internet Key Exchange) 프로토콜의 키 교환 및 SA 협상 과정
* SSL VPN의 특징과 IPSec VPN과의 장단점 비교

### [정보보안기사] 2-13. 라우터 보안 설정과 ACL 작성법
* Cisco 라우터 접근 제어: Standard ACL vs Extended ACL 문법
* Ingress 필터링(유입 차단)과 Egress 필터링(유출 차단)의 개념
* 라우터 패스워드 암호화 설정(enable secret) 및 원격 접속 보안
* 출발지/목적지 IP와 포트를 기반으로 한 ACL 차단 룰 작성 실습

## Part 3. 애플리케이션 보안 (Application Security)

### [정보보안기사] 3-1. HTTP 헤더 분석과 주요 메소드 보안
* HTTP 요청/응답 헤더 상세 분석 (User-Agent, Referer, Cookie, Server)

### [정보보안기사] 3-2. 웹 서버(Apache) 보안 설정과 로그 분석

### [정보보안기사] 3-3. FTP(Active/Passive)와 이메일 보안

* 이메일 보안 프로토콜: PGP, S/MIME, PEM 특징 비교
* 스팸 메일 차단 기술: SPF(IP 확인), DKIM(전자서명), DMARC(정책)

### [정보보안기사] 3-4. SQL Injection 공격 원리 및 시큐어 코딩

* Blind SQL Injection에서 참/거짓 판별 방법과 자동화 도구 개념
* 대응 방안: 입력값 검증 및 Prepared Statement(Precompiled Query) 사용

### [정보보안기사] 3-5. XSS와 CSRF 공격 차이점 및 방어법
* XSS(Cross Site Scripting): Reflected, Stored, DOM 기반 공격 원리
* CSRF(Cross Site Request Forgery): 권한 도용 및 요청 위조 메커니즘
* XSS 방어를 위한 입출력 필터링 및 쿠키 HttpOnly 속성 설정
* CSRF 방어를 위한 Referer 검증 및 CSRF Token 활용법

### [정보보안기사] 3-6. 파일 업로드 취약점(Web Shell) 대응
* 파일 업로드 기능을 악용한 웹쉘(Web Shell) 업로드 및 원격 제어 실습
* 확장자 우회 기법(Null Byte, 대소문자, 이중 확장자)과 대응

* 파일 업로드 시 무작위 파일명 변경 및 저장 경로 은폐 전략

### [정보보안기사] 3-7. 디렉토리 리스팅 및 파라미터 변조 공격
* 디렉토리 리스팅 취약점 확인 방법 및 구글 해킹(Google Dork)
* 웹 파라미터 변조를 통한 가격 조작, 타인 게시글 삭제 공격 실습
* 입력값 유효성 검증과 세션 검증을 통한 파라미터 변조 방어
* 불충분한 인증 및 인가 취약점 사례 분석

### [정보보안기사] 3-8. DB 보안 3요소와 무결성 확보 방안
* 데이터베이스 보안 요구사항: 접근통제, 추론 방지, 무결성 보장
* 무결성 제약조건 3가지: 개체 무결성, 참조 무결성, 도메인 무결성

## Part 4. 정보보안 일반 (General Security)

### [정보보안기사] 4-1. 암호학 개요와 보안 5대 원칙
* 보안의 5대 원칙: 기밀성, 무결성, 가용성, 인증, 부인방지
* 암호 해독(Cryptanalysis) 공격 유형: COA, KPA, CPA, CCA 차이점
* 스테가노그래피(Steganography)와 워터마킹의 개념
* 커크호프의 원리(Kerckhoffs's Principle)와 알고리즘 공개의 중요성

### [정보보안기사] 4-2. 대칭키 암호(AES, DES)와 운영 모드
* 대칭키 암호 알고리즘 종류: DES, 3DES, AES, SEED, ARIA, HIGHT, LEA
* 블록 암호 운영 모드 비교: ECB, CBC, CFB, OFB, CTR 특징
* ECB 모드의 패턴 노출 취약점(펭귄 그림 예시)과 CBC 모드의 IV 활용
* AES 알고리즘의 블록 크기(128bit)와 키 길이(128/192/256bit) 스펙

### [정보보안기사] 4-3. 공개키 암호(RSA)와 키 배송 문제
* 대칭키 암호의 키 배송 문제 해결을 위한 공개키 암호의 탄생
* RSA(소인수분해), ElGamal(이산대수), ECC(타원곡선 이산대수) 기반 문제
* 하이브리드 암호 시스템: 대칭키로 데이터 암호화 + 공개키로 키 암호화
* RSA 키 생성 과정의 기초 수학적 원리(오일러 함수 등)

### [정보보안기사] 4-4. 해시 함수(SHA)와 메시지 인증(MAC)
* 일방향 해시 함수의 성질: 역상 저항성, 제2 역상 저항성, 충돌 저항성
* 주요 해시 알고리즘: MD5(취약), SHA-1(중단), SHA-256/512, HAS-160
* MAC(Message Authentication Code)의 개념: 무결성 + 인증 보장
* 해시 함수와 MAC의 차이점 및 HMAC 동작 방식

### [정보보안기사] 4-5. PKI 구조와 전자서명 동작 원리
* 전자서명 프로세스: 개인키로 서명(암호화) -> 공개키로 검증(복호화)
* PKI(공개키 기반 구조) 구성요소: CA(인증기관), RA(등록기관), Repository
* X.509 인증서 구조 및 CRL(인증서 폐기 목록) 확인 방법
* OCSP(온라인 인증서 상태 프로토콜)와 CRL의 실시간성 비교

### [정보보안기사] 4-6. 접근 통제 모델: DAC, MAC, RBAC
* 접근 통제 3단계: 식별(Identification), 인증(Authentication), 인가(Authorization)
* DAC(신분 기반), MAC(보안 등급 기반), RBAC(역할 기반) 모델 상세 비교
* 벨-라파듈라(BLP) 모델(기밀성) vs 비바(Biba) 모델(무결성) 규칙 비교
* 접근 통제 행렬(ACM), ACL(객체 관점), CL(주체 관점)의 차이

### [정보보안기사] 4-7. 사용자 인증: SSO, Kerberos, OTP
* SSO(Single Sign On)의 편의성과 보안 위협(단일 실패 지점)
* Kerberos 인증 프로토콜 구조: KDC, AS, TGS, Ticket, Authenticator
* 타임스탬프를 이용한 Kerberos의 재전송 공격 방지 원리
* OTP(One Time Password) 생성 방식(시간 동기화/이벤트 동기화)과 FIDO

## Part 6. 실기 Special (종합 대응) - [최신 트렌드]

### [정보보안기사] 6-1. 침해 사고 대응 절차와 포렌식 원칙
* KISA 침해 사고 대응 7단계: 사고전 준비 ~ 사후 분석 프로세스 암기
* 디지털 포렌식 5대 원칙: 정당성, 재현성, 신속성, 연계 보관성, 무결성
* Chain of Custody(연계 보관성) 절차 및 증거물 무결성 입증(Hash)
* 리눅스 dd 명령어를 이용한 디스크 이미징 및 md5sum 해시값 비교 실습

### [정보보안기사] 6-2. [신기술] 제로 트러스트, AI 보안, 공급망 보안
* 제로 트러스트(Zero Trust) 가이드라인 1.0 핵심: "Never Trust, Always Verify"
* 공급망 보안(Supply Chain Security)과 SBOM(SW 자재 명세서)의 필요성
* AI 보안: LLM(거대언어모델) 취약점(Prompt Injection)과 데이터 오염 위협
* 클라우드 보안 용어: SecaaS, CASB, SOAR(보안 오케스트레이션) 개념 정리
