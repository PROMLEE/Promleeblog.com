## 들어가기 전에
블로그 개발을 위한 기본 설정을 해보도록 하자.
이 포스팅에서 설정할 내용은 다음과 같다.

- Next.js 프로젝트 생성
- Tailwind CSS(Shadcn/ui) 설정

<Bbox>Node.js 18버전 이상, npm이 설치되어 있어야 한다.</Bbox>

## Next.js 프로젝트 생성 및 실행
### 프로젝트 생성
Next.js 프로젝트를 생성하기 위해 다음 명령어를 실행한다.
```bash
npx create-next-app@latest
```
필자는 모두 기본 설정으로 진행하였다.
<Img src="/posts/240709-131008.png" width={500}/>
1. **프로젝트 이름**을 입력한다.
2. **TypeScript**를 사용할 것인지 선택한다.
	- JavaScript를 사용할 것이라면 **No**를 선택한다.
3. **ESLint**를 사용할 것인지 선택한다.
4. **Tailwind CSS**를 사용할 것인지 선택한다.
5. **'src/'** 디렉토리를 생성할 것인지 선택한다.
6. **App 라우팅**을 사용할 것인지 선택한다.(Next.js 13 버전 이상 지원)
	- Page 라우팅을 사용할 것이라면 **No**를 선택한다.
7. **절대 경로 설정**을 수정할 것인지 선택한다.
	- 기본 설정은 **'src/'** 디렉토리이다.

### 프로젝트 실행
프로젝트 생성이 완료되면 다음 명령어를 실행하여 추가 패키지를 최신 버전으로 설치(업데이트)한다.
```bash
cd {프로젝트 이름}
npm install next@latest react@latest react-dom@latest
```
프로젝트를 실행하기 위해 다음 명령어를 실행한다.
```bash
npm run dev
```
프로젝트가 정상적으로 실행 후 브라우저에서 http://localhost:3000 으로 접속하면 다음과 같은 화면이 나타난다.
<Img src="/posts/240709-133009.png" width={2000}/>
참고: 해당 화면은 `src/app/page.tsx` 파일을 렌더링한 페이지이다.

## Tailwind CSS 설정
### Shadcn/ui 설치 (선택)
Shadcn/ui는 Tailwind CSS를 사용한 UI 라이브러리이다.
[Shadcn/ui](https://ui.shadcn.com)
다음 명령어를 실행하여 Shadcn/ui를 설치한다.
tailwind-merge는 Shadcn/ui에서 사용하는 플러그인이다.
```bash
npx shadcn-ui@latest init
npm i tailwind-merge
```
기본 설정으로 진행하였다.
<Img src="/posts/240709-135300.png" width={500} alt="안녕"/>


### tailwindcss/typography 설치 (선택)
기본적으로 Next.js 프로젝트 생성 시 Tailwind CSS 설정이 완료되어 있다.
다만, Tailwind CSS는 HTML 태그의 스타일을 무시하는 특성이 있어, 후에 Markdown 파일을 렌더링할 때 스타일이 적용되지 않는다.
이를 해결하기 위한 것이 `prose` 옵션이다.
`prose` 옵션은 `tailwindcss/typography` 플러그인을 사용하여 Markdown 파일을 렌더링할 때 스타일을 적용한다.
다음 명령어를 실행하여 `tailwindcss/typography` 플러그인을 설치한다.
```bash
npm i @tailwindcss/typography
```
tailwind.config.ts 파일을 열어 다음과 같이 플러그인을 추가한다.
```js title="tailwind.config.ts"
module.exports = {
	// ...
	plugins: [
		require('@tailwindcss/typography'),
		// ...
	],
	// ...
}
```
